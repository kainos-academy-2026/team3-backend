# Ticket Implementation Files

This folder contains one implementation document per ticket.

## Naming
- `TICKET-<id>.md` (preferred)
- `TICKET-<short-slug>.md` (when no ID exists)

## Workflow
1. Copy `TICKET_TEMPLATE.md` to a new ticket file.
2. Fill in scope, plan, and expected file changes before coding.
3. Update implementation notes while coding.
4. Record tests and final status before merge.

## Example
- `TICKET-123.md`
- `TICKET-auth-login-rate-limit.md`
